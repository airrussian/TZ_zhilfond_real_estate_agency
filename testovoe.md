# **Тестовое задание**

Backend Developer · Middle

PHP · Laravel · Архитектура

## **Контекст**

Вам необходимо реализовать небольшой сервис уведомлений. Представьте, что это часть реальной системы — пользователи должны получать уведомления через разные каналы, а отчётность по ним должна быть доступна для анализа.

Задание намеренно оставляет некоторые архитектурные решения за вами. Нас интересует не только работающий код, но и то, как вы принимаете решения и объясняете их в README.

## **Основное задание**

### **1\. Уведомление**

Уведомление должно содержать:

* Текст сообщения (не более 500 символов)  
* Статус: в обработке, отправлено, ошибка  
* ID пользователя-получателя

### **2\. API**

Необходимо реализовать REST API для работы с уведомлениями. API должен позволять создавать уведомление, получать его статус, а также просматривать историю уведомлений пользователя с фильтрацией по статусу и каналу.

### **3\. Каналы отправки**

Поддержите два канала: email и telegram. Реальную отправку реализовывать не нужно — достаточно заглушек. Важно: система должна быть спроектирована так, чтобы добавление нового канала в будущем не требовало изменения существующего кода.

### **4\. Гарантия доставки**

Доставка каждого уведомления должна быть гарантирована. Продумайте, что должно происходить при сбое: как система поймёт, что уведомление не было доставлено, и как обеспечит повторную попытку. Выбор механизма остаётся за вами.

### **Дополнительное задание (необязательно)**

Реализуйте генерацию отчёта по уведомлениям для пользователя. Отчёт должен отражать:

* Количество уведомлений по каждому каналу  
* Количество ошибок по каждому каналу

Готовый файл сохраняйте во внутреннюю файловую систему. API должен позволять запросить генерацию отчёта за период, узнать статус готовности и скачать файл. Продумайте, что должно происходить, если генерация упадёт на середине.

## **Технические требования**

| Требование | Детали |
| :---- | :---- |
| **Стек** | Laravel 11+, PHP 8.1+ |
| **База данных** | MySQL или Pgsql |
| **Сборка** | Docker или docker-compose — полностью |
| **Валидация** | FormRequest для входящих данных |

## **Стандарты разработки**

Следующие требования являются обязательными.

## **Тесты**

Должны быть написаны функциональные и unit тесты. Тесты должны покрывать бизнес-функционал: создание уведомления, смену статусов, обработку ошибок. 100% покрытия не требуется.

## **Статический анализ**

В проекте должен быть настроен PHPStan с минимальным уровнем 5\. Проверка должна проходить без ошибок на момент сдачи.

## **Code style**

Должен быть подключён и настроен code style fixer — Pint, PHP CS Fixer или аналог. Выбор инструмента и правил остаётся за вами, важно что они зафиксированы в конфиге и применяются к коду.
